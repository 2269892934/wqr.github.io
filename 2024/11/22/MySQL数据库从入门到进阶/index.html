MySQL数据库学习记录

MySQL简介

数据库相关概念

  1. 数据库服务器：运行数据库管理软件的计算机
  2. 数据库管理软件：mysql, oracle, db2, sqlserver
  3. 库：文件夹
  4. 表：文件
  5. 记录：事物一系列典型的特征：Nancy,female,18
  6. 数据：描述事物特征的符号

MySQL介绍

  1. 关系型数据库管理系统
  2. 目前属于Oracle旗下公司，由瑞典MySQL AB公司开发
  3. 最流行
  4. 基于socket编写的C/S架构的软件
  5. 客户端软件
    mysql自带：如mysql命令，mysqldump命令等
    python模块：如pymysql

数据库管理软件

分为两大类：

  • 关系型：如sqllite, db2, oracle, access, sql, server, MySQL,注意：sql语句通用
  • 非关系型：mongodb, redis, memcache

可以简单理解为：

  • 关系型数据库需要有表结构
  • 非关系型数据库是key-value存储的，没有表结构

下载安装

Windows平台安装mysql
  1. 官网下载安装包, DOWNLOADS下的社区版Community(MySQL Community /server 5.7.16)
  2. ZIP压缩包解压完就安装好了
  3. 设置环境变量 Path里添加mysql安装路径**/bin
  4. data文件夹：数据库里的文件夹和文件（库和表）
  5. 启动mysql 命令：mysql，每次开机启动麻烦，制作一个系统服务
  6. 杀死mysql进程 查询进程: tasklist | findstr mysql 杀死进程: taskkill /F /PID 进程号
  7. 制作系统服务：mysqld --install, 可能会有权限拒绝，记得以管理员身份打开命令行
  8. 解除系统服务：mysqld --remove
  9. 制作启动服务后，在服务中就可以找到MySQl的启动方式是自启动的
  10. 密码设置与破解
    • 查询mysql当前登录账号： 1. 执行mysql,进入mysql 2. 执行命令 select user(); 默认登录的不是管理员账号
    • 切换登录账号： 1. 先退出mysql exit 2. 执行命令 mysql -uroot -p （默认用户没有密码），回车，提示输入密码，默认无密码，所以再回车
    • 设置管理员密码: 1. 先退出mysql exit 2. 执行命令 mysqladmin -uroot -p password "123456",回车，提示输入密码，默认无密码，所以再回车
    • 登录管理员账户，这次需要输入密码： 执行命令 mysql -uroot -p123456
    • 设置密码后，再想修改密码：执行命令 mysqladmin -uroot -p123456 password "456789"
    • 忘记密码怎么办？ 1. 关闭mysql，以管理员身份启动终端， 执行net stop mysql服务名，也可以在服务里面关闭 2. 跳过授权表启动mysql: mysqld --skip-grant-tables,再启动一个终端，执行mysql -uroot -p,此时不用输入密码也可以登录进去 3. 修改密码，登录进去后，执行命令update mysql.user set password=password("你的密码") where user="root" and host="localhost";，password("你的密码"),这样设置的密码不是明文形式 4. 刷新授权表 flush privileges; 5. 退出mysql，exit, 由于之前是跳过授权表启动mysql，所以要先杀死之前的mysql进程（以管理员身份启动终端，执行上面的第6步），再正常启动（见上面第5步），再正常登录进去就行mysql -uroot -p你的密码
    • 套接字服务端默认值 mysql -uroot -p123456 -h 127.0.0.1 -P 3306（远程账号的使用，后面讲）

初识SQL语句

SQL语句分为三大类：

  1. DDL语句 数据库定义语言：数据库、表、视图、索引、存储过程，例如：CREATE DROP ALTER
  2. DML语句 数据库操纵语言：插入数据INSERT、删除数据DELETE、更新数据UPDATE、查询数据SELECT
  3. DCL语句 数据库控制语言：例如控制用户的访问权限GRANT、REVOKE

数据类型
数值类型

  1. 整数类型
    TINYINT SMALLINT MEDIUMINT INT BIGINT
    作用：存储年龄，等级，id，各种号码等

    • tinyint 1字节 8byte位 小整数，数据类型用于保存一些范围的整数数值范围 有符号： -128127 无符号：0255 PS: MySQL中无布尔值，使用tinyint(1)构造
    • smallint 2字节 大整数，数据类型用于保存一些范围的整数数值范围 有符号： -3276832767 无符号：065535
    • mediumint 3字节 大整数，数据类型用于保存一些范围的整数数值范围 有符号： -83886088388607 无符号：016777215
    • int/integer 4字节 大整数，数据类型用以保存一些范围的整数数值范围 有符号： -21474836482147483647 无符号：04294967295
    • bigint 8字节 极大整数，数据类型用以保存一些范围的整数数值范围 有符号： -9223372036854775808~9223372036854775807 无符号：18446744073709551615
      创建无符号整形，添加约束条件：unsigned
      int的字节已经定死，int(宽度)里面的宽度指的是显示宽度，就算设置了，当数值过大就会显示超过宽度的数值，所以用默认的行，宽度默认最大的值的宽度
      zerofill 约束条件，用0填充

  2. 浮点类型
    定点数类型 DEC等同于DECIMAL
    FLOAT DOUBLE
    作用：存储薪资、身高、体重、体质参数等，工资没必要用decimal

  • float 单精度浮点数 随着小数的增多，精度变得不准确 float(m, d) m是字数总个数，d是小数点后个数，m最大值为255，d最大值为30
  • double 双精度浮点数 随着小数的增多，精度比float要高，但也会变得不准确 double(m, d) m是字数总个数，d是小数点后个数，m最大值为255，d最大值为30
  • decimal 准确的小数值 随着小数的增多，精度始终准确对于精确数值计算时需要用此类型，decimal能够存储精确值的原因在于其内部按照字符串存储 decimal(m, [,d]) m是字数总个数，d是小数点后个数，m最大值为65，d最大值为30
日期类型

year date time datetime timestamp
1997 1997-10-01 12:12:12 1997-10-01 12:12:12 875707932

datetime和timestamp存的都是年月日时分秒，区别是：

  • datetime的日期范围是1001–9999年，timestamp的时间范围是1970–2038年
  • datetime存储时间与时区无关，timestamp存储时间与时区有关，显示的值也依赖于时区，在mysql服务器，操作系统以及客户端连接都有时区的设置
  • datetime使用8字节的存储空间，timestamp的存储空间为4字节，因此，timestamp比datetime的空间利用率高
  • datetime的默认值为null，timestamp的字段默认不为空，默认为当前时间CURRENT_TIMESTAMP，如果没有特殊处理，update更新语句中没有该字段的值的话默认当前时间
字符串类型
  1. char: 定长 传的值长度不够会在末尾补上空格,本身末尾有空格，查的时候也会去掉 获取长度会给你把空格删了（只针对末尾）
  2. varchar: 变长 传几个就是几个 获取正确的长度 1bytes + “”,会多存一位

宽度指字符的个数 char(5) varchar(5)

略施小计，让char现出原形
mysql> SET sql_mode = 'PAD_CHAR_TO_FULL_LENGTH';

char类型数据 查询时末尾有再多空格都会被忽略，前面有空格的话不会
模糊查询：where name like “值”，这种情况下，不会忽略空格

枚举类型和集合类型

字段的值只能在给定范围中选择，如单选框、多选框
enum 单选，只能在给定范围内选一个值，如性别sex male/female sex enum('male','female')
set 多选，在给定范围内可以选择一个或一个以上的值（爱好1，爱好2，爱好3…）hobbies set('read','music','run')

  3. 约束条件 primary key 主键，不为空且唯一 not null unique

  4. 约束条件auto_increment 自增长

    show variables like 'auto_inc%' 查看当前数据库的自增长设置 了解就行
    步长：auto_increment_increment默认为1
    起始偏移量：auto_increment_offset默认为1

    • group by 分组
      a. 首先, 分组发生在where之后。即分组是基于where之后得到的记录而进行的，没有where也可以，但有的话必须在group之前
      b. 分组指的是，将所有记录按照某个相同字段进行归类。例如：性别，职位等
      严格分组模式：set global sql_mode="ONLY_FULL_GROUP_BY" 分组只能取分组的字段
      select post from employee group by post;
      c. 为何要分组？ 取每个部门的最高工资 取每个部门的员工数 取男人数和女人数
      d. 大前提：可以按照任意字段分组，但是分组完毕后，比如group by post,只能查看post字段，如果想看组内信息，需要借助于聚合函数

      强调： 不能用unique字段作为分组的依据，每条记录自成一组，没有意义，多交记录之间的某个字段值相同，该字段通常用来作为分组的依据

      聚合函数 max min avg sum count

      • 每个部门有多少员工
        select post,count(id) as emp_count from employee group by post
      • 每个部门最高工资
        select post,max(salary) as max_salary from employee group by post
      • 每个部门最低工资
        select post,min(salary) as min_salary from employee group by post
      • 每个部门平均工资
        select post,avg(salary) as avg_salary from employee group by post
      • 每个部门工资求和
        select post,sum(salary) as sum_salary from employee group by post
      • 每个部门员工的名字 group_concat
        select post,group_concat(name) from employee group by post

      强调： 聚合函数聚合的是组的内容，若是没有分组，则默认一组

    • having 过滤
      分组之后的过滤
      !!! 执行优先级顺序 where > group by > having

      1. where发生在分组group by之前，因而where中可以有任意字段，但是绝对不能使用聚合函数
      2. having发生在分组group by之后，因而having中可以使用分组的字段，无法直接取到其他字段，可以使用聚合函数

      优先级 from > where > group by > having > distinct > order by > limit
